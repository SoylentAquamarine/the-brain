Architecture — The Brain

Blueprint · Last updated 2026-05-02

Module Map

ModulePurpose
delegate.pyCLI entry point — parses --provider, --type, --prompt, --context flags; builds a Task; invokes the orchestrator; prints the result.
brain/orchestrator.pyCore dispatcher — runs a Task through the router and chosen adapter; retries on failure using the per-type fallback chain.
brain/router.pyRouting table plus per-task_type ordered fallback chains; maps each task type to a list of provider keys to try in sequence.
brain/stats.pyAppends every call's metadata to stats/usage.json (provider, model, tokens, latency, cost, success, error) regardless of outcome.
brain/adapters/One file per provider — each exports async def call(task: Task) -> Result.
mcp_server.pyMCP server — exposes the delegate tool natively in Claude Code; same orchestrator, no bash required.
session_start.pySession startup script — runs health check, prints provider status table, emits routing advisories.
health_check.pyPolls each provider with a lightweight test call; reports online/offline/degraded status.
report.pyFormats and prints a usage summary from stats/usage.json.
push_stats.pyUpdates the GitHub README stats table and pushes to GitHub.

Provider Adapters

One file per provider in brain/adapters/:

  • cerebras.py, groq.py, gemini.py, mistral.py, sambanova.py
  • fireworks.py, openai.py, cohere.py, cloudflare.py
  • huggingface.py, openrouter.py, ollama.py, pollinations.py

Adapter Contract

Every adapter implements the same interface:

async def call(task: Task) -> Result

Result fields:

FieldTypeDescription
providerstrProvider key used
modelstrModel identifier
contentstrAI response text
tokens_usedintOutput token count
latency_msintEnd-to-end response time
cost_usdfloatEstimated cost (0.0 for free tier)
succeededboolWhether the call succeeded
errorstr | NoneError message if failed

Task Object

FieldTypeDescription
promptstrThe input prompt
task_typeTaskType enumRouting category (classification, coding, reasoning, etc.)
contextstr | NoneOptional text prepended to the prompt (document content, file contents)
max_tokensintMaximum output tokens (default 2048)
preferred_modelstrProvider key to try first

Fallback Chain Execution

  1. Orchestrator invokes the adapter for preferred_model.
  2. On failure, router looks up the fallback chain for task_type.
  3. Orchestrator iterates the chain, trying each provider until one succeeds.
  4. All attempts are logged to stats/usage.json — including failures.
  5. If no provider succeeds, an error Result is returned.
