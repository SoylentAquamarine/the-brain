Health System — The Brain

Blueprint · Last updated 2026-05-02

Overview

health_check.py polls provider endpoints to verify they are reachable and responding before tasks are delegated. Results are informational — used by session_start.py to surface provider status at session startup so routing decisions can be made with current information.

What Is Checked

Each provider is tested with a lightweight API call (a short prompt). Success is defined as HTTP 2xx with a valid response body. Failure includes HTTP errors (4xx, 5xx), timeouts, and malformed responses.

Output Columns

ColumnDescription
providerProvider key
statusonline, offline, or degraded
latency_msResponse time of the health check call
modelModel used for the test call
errorError message if status is not online

session_start.py Integration

session_start.py runs health_check.py at the start of every Claude Code session. It prints a provider status table to the terminal and emits routing advisories if a primary provider is offline — so Claude can adjust delegate.py calls before any work begins.

Advisory-Only — No Automatic Penalty

Unlike ManyAI Desktop, The Brain's health system does not automatically penalize offline providers in routing. It is advisory only. Claude reads the advisories and adjusts provider choices manually. This keeps the routing logic simple and fully transparent.

Manual Invocation

python health_check.py

Run standalone at any time to check current provider status without starting a full session.
