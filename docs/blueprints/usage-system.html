Usage System — The Brain

Blueprint · Last updated 2026-05-02

Overview

Every delegate.py call is appended to stats/usage.json regardless of success or failure. This provides a persistent record of all AI worker usage for cost analysis and routing optimization.

stats/usage.json Schema

{
  "ts": "2026-05-02T14:23:11",
  "provider": "mistral",
  "model": "mistral-small-latest",
  "task_type": "coding",
  "tokens_used": 512,
  "latency_ms": 1172,
  "cost_usd": 0.000000,
  "succeeded": true,
  "error": null
}

The file is an append-only JSON array, auto-created on first call, and gitignored (not committed to GitHub).

Key Metrics Tracked

MetricDescription
tsISO 8601 timestamp of the call
provider / modelWhich provider and model handled the task
task_typeRouting category — enables per-type breakdowns
tokens_usedOutput token count from provider response
latency_msEnd-to-end response time
cost_usdProvider cost (0.0 for free tier providers)
succeededWhether the call returned a valid response
errorError message for failed calls, null on success

report.py

Run python report.py for a formatted usage summary showing:

  • Per-provider breakdown: calls, tokens, average latency, success rate, total cost
  • Overall totals
  • Estimated Claude tokens saved

push_stats.py

Run python push_stats.py to update the GitHub README stats table with the latest provider metrics and push to GitHub. Run at the end of each session.
