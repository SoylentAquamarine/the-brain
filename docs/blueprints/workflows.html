Workflows — The Brain

Blueprint · Last updated 2026-05-02

No Workflow System

The Brain does not have a workflow system in the traditional sense. There are no tabs, sessions, or persistent conversation contexts. Each delegate.py call is stateless and independent. The equivalent of a workflow is a task type — the routing category that determines which provider chain is used.

Task Types

Task TypeDescriptionPrimary Provider
classificationYes/no, scoring, labellingcerebras
factual_qaQuick factual questions and general explanationcerebras / groq
summarizationLong text compression, up to 1M token contextgemini
translationAny language pairgemini
codingCode generation and debuggingmistral
extractionStructured data extraction from existing textmistral
creativeWriting, drafting, brainstormingmistral
reasoningDeep analysis and trade-off evaluationsambanova
generalAnything that does not fit another typecerebras

How Task Type Affects Routing

router.py maintains a full fallback chain per task type. The primary provider is first; if it fails the next is tried automatically. The chain is tuned so the most capable provider for each category is attempted first.

Choosing the Right Task Type

  • Use classification for binary answers, factual_qa for explanations — not interchangeable.
  • Use extraction (not coding) when pulling structured data from existing text.
  • Use reasoning only for genuinely complex multi-step analysis — SambaNova has higher latency than other primary providers.
  • When uncertain, general routes to cerebras first and falls back broadly.
